To decide on a suitable range of media to design for, I start with viewport width. I discovered the following to be the most popular:

  1. 320x568
  2. 360x640
  3. 375x667
  4. 414x736
  5. 768x1024
  6. 1366x768

Some dimensions are considered standard, such as an iPad (768x1024), or a typical Android device (360x640).

With the dimensions I create what I refer to as a Container Grid (Fig. 1-4).

A Container Grid is a container, yet only constructed using CSS, not a range of <div> wrappers, although it can also be applied to that kind of structure. It's main purpose is to act as an overall guide that makes it easy to decide where to place content.
Between breakpoints, any content within the fluid-margin will have a fluid width, reflowing with the size of the viewport, whilst content placed across both margins will adhere absolutely to the width of a container.
If a viewport is wider than all columns, gutters and margins, fluid-margin can be utilised to solve a range of design problems (Fig. 4).

Making use of CSS Grid and viewport units means the layout is not described by markup.

Combining Grid Template Areas, Media Queries and Custom Properties make the process of spanning different ranges of columns almost trivial (Fig. 6).

You can span all columns, across breakpoints without using media queries to re-declare in each breakpoint how many columns to span.
