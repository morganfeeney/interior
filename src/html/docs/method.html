Using the grid

As I detailed in this article, what I learnt from the first incarnation of Interior — that was based on Bootstrap — is I had to walk away from what came before, and embrace new CSS layout tools. Interior v.2 doesn't rely on floats for layout, it never will, I just need to create a grid and gutters, flexbox does the rest. So that's how Interior v2. works; you won't find any .col-[whatever] classes or .push, .pull or .offset classes like you get in Bootstrap... well you might but if they exist they'll be for baseline alignment.

Columns & gutters

In an effort to reduce the amount of markup necessary to create columns with gutters I opted for the use of margin over padding. This means to create a gutter and to apply a background to the content within it that is padded, so you can actually see the gutter, this means less code.

Separation

Keeping markup and presentation separate makes life easier going forward. There are many benefits, the primary benefit being that changes can be made using only CSS.
