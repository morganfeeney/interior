Web layout design is evolving.

Something big is happening, specificaton is pushing the web forward in an unprecedented way. Those ready to embrace change are on the verge of a new era. However, we mustn't forget that beyond the web is a wealth of layout design resource to learn from.

Web and print are different beasts

Paper can fold yet it doesn't shrink. You needn't worry about a paper layout bursting at the seams when it is rotated. The fluidity of web alone presents a unique set of challenges, the world of print won't experience.

The transition/translation from print to web is challenging, you can't apply that 'pretty' layout you created in Illustrator to the browser, whilst solving issues surounding content reflow. Besides... we've been limited by browser capability, we still are but we're moving forward.

The capability of browsers has lead to unreliable hacks, javascript frameworks for presentation, and float based grid systems for scaffolding. They are workarounds, are problematic, and will eventually break if you push (or scroll) too hard. We've wanted to be able to create those lovely designs yet until now haven't had the tools.

Time to see what the shiny new toys can do

It's so important that we move on from how things have been. In order to do this we need direction. People often draw a line between web and print, I see the line becoming blurred.

We can use some great examples of print layouts and see how far they can go when applied with the new features that we have.

The need for Interior

Automation, structure, re-usable & familiar code. I wanted a framework that suits my needs, it's designed for me. I always end up customising things in the end so thought it was time to bite the bullet. Feel free to use it & get involved if you feel strongly about contributing.
