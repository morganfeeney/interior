Separation

Keeping markup and presentation separate makes life easier going forward. There are many benefits, the main one being that changes can be made via CSS only.

Example

Think about it... you have a layout, you want to change it, you can either change both HTML and CSS or change CSS only. This means you are not limited by the markup. For example you have series of columns inside a container, when you created the first design those columns were lined up in rows of 3. You then decide to change that layout so that there are only 2 columns. If you keep your markup generic this change could be a matter of updating the CSS only.

This is a great approach to use when coding layouts in the browser using livereload to refresh when you make changes.
