Why?

The Baseline Grid and Modular Scale go hand-in-hand, for instance; the baseline grid is used to visually align text by indicating the location of the baseline, its proportions are created using the same proportions the text styles are created with.

Interior makes full use of the grid by altering it’s proportions using percentages on the HTML element. This way a single rem unit (the measurement used for incrementally traversing the baseline grid) will be relative to the percentage inherited throughout the entire document. It eventually makes life easy.

If you size HTML as follows:


75%

87.5%

112.5%

…you get the following base font-sizes / line-heights to work with:


12px

14px

18px

…therefore a simple rem becomes incredibly useful.

It also means that line-height must always be in units based upon the initial value of 1rem (if that’s your baseline), but getting the darn thing to sit on the Baseline Grid is another story.

Vertical rhythm

Everything I’ve learnt recently about typography leads to units of measurement being the fundamental building blocks of order and harmony in design. A Baseline Grid can be used as a visual aid to enable this harmony by highlighting where text can fit into a layout.

But surely we can already do this with CSS?

… yes we can, but when we start to piece the various components of a web page together as part of a system, and want to align things vertically (horizontally isn’t a problem), the current spec is flawed[possibly revise wording here]. Process

Pick a font-stack

I use Helvetica & system fonts quite a lot so didn't need to make too many adjustments to the default cap-height settings in Interior. If you pick different fonts you’ll need to work out what the cap-height is and apply it. if you plan to use Interior there’s a variable for cap-height and it’s tied into the baseline mixin.

Establish a unit for the baseline

I use 1rem (see above, link to HTML sizing part), as it works really well for several reasons—even though it has already been prescribed by Sassline (and others), I like to make life difficult for myself and went through the process of completely ignoring it by using unit-less values, in the end the way a browser renders unit-less values over unit values (e.g. rems), resulted in computed sizes that were incorrect [provide example, codepen] there were some severe limitations. Thankfully Sassline was available as a reference.

Make adjustments

This is the tricky—and possibly most time consuming—part, and what the visual baseline grid is for. To align text to “sit” on the baseline I need to see things in action, so I open up a browser window and view text on a baseline grid, if it doesn’t quite sit right I make adjustments until it does.

What I found of most use to get this correct was creating sample pages with varying degrees of content, paragraphs, headings, block-quotes, citations, images, figures with captions etc. then adding and removing things-each element impacts the next one down the baseline (unless you use an explicit grid to split up each element).
