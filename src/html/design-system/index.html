Whilst using Interior, and simultaneously writing up how it should be used, it occurred to me that I can’t always keep up with the development of new specifications. Not only that but what if I want to switch methodologies? Apply helper classes one day or the separation of concerns on another? Why tie thing down? Therefore separating the fundamental design system—which can apply to a design created with any spec—from the ever-changing is not only logical, but efficient.
